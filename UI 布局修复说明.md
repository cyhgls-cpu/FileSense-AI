## 💡 技术解释

### 为什么会出现这个 bug？

**模板字符串嵌套** 是 Electron/前端开发中容易出错的地方：

```javascript
// 错误示例
html += `
  <div class="card">
    ${condition ? `<div>${value}</div>'}</div>` : ''}
                                ↑ 多余的引号和标签
  </div>
`;

// 正确示例
html += `
  <div class="card">
    ${condition ? `<div>${value}</div>` : ''}
  </div>
`;
```

**问题根源**:
1. 字符串拼接时多加了 `</div>`
2. 引号位置错误导致 HTML 结构错乱
3. 浏览器解析时自动闭合标签，导致布局错位

### Grid 布局的关键属性

```css
.duplicate-compare {
  display: grid;              /* 启用 Grid 布局 */
  grid-template-columns: 1fr auto 1fr; /* 左中右三列 */
  gap: 15px;                  /* 间距 */
  align-items: stretch;       /* 拉伸对齐 */
  min-width: 0;               /* 关键：允许缩小 */
  width: 100%;                /* 占满容器 */
}
```

**每个属性的作用**:
- `display: grid` - 启用网格布局
- `grid-template-columns: 1fr auto 1fr` - 左右等宽，中间自适应
- `gap: 15px` - 卡片间距
- `align-items: stretch` - 所有卡片高度一致
- `min-width: 0` - 允许 grid 项目缩小（防止溢出）
- `width: 100%` - 确保占满父容器
