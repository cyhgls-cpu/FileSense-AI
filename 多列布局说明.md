# 📐 重复文件对比组多列布局说明

## ✨ 新功能

重复文件对比组现在支持**自适应多列显示**！窗口越大，显示的列数越多。

---

## 🎯 布局规则

### 响应式列数

| 窗口宽度 | 列数 | 每列宽度 | 效果 |
|---------|------|---------|------|
| < 1200px | 1 列 | 100% | 单列垂直排列 |
| 1200-1600px | 自动 | 600px | 根据宽度自动调整 |
| 1600-2000px | 自动 | 650px | 更宽的列 |
| > 2000px | 自动 | 700px | 超宽屏优化 |

### 计算公式

```
列数 = Math.floor(窗口宽度 / 600px)
```

**示例**:
- 1200px 宽度 → 2 列
- 1800px 宽度 → 3 列
- 2400px 宽度 → 4 列
- 3000px 宽度 → 5 列

## 🔧 技术实现

### CSS Grid 布局

```css
.file-list {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(600px, 1fr));
  gap: 20px;
  align-content: start;
}
```

**属性解释**:
- `display: grid` - 启用网格布局
- `repeat(auto-fill, ...)` - 自动填充列数
- `minmax(600px, 1fr)` - 每列最小 600px，最大占满
- `gap: 20px` - 卡片间距
- `align-content: start` - 从顶部开始排列

### 工作原理

1. **浏览器计算**: 
   ```
   可用宽度 = 窗口宽度 - 边距
   列数 = floor(可用宽度 / 600px)
   每列实际宽度 = 可用宽度 / 列数
   ```

2. **自动调整**:
   - 窗口变宽 → 列数增加
   - 窗口变窄 → 列数减少
   - 无需刷新，实时响应

3. **卡片自适应**:
   ```css
   .duplicate-group {
     width: 100%;      /* 占满单元格 */
     min-width: 0;     /* 允许缩小 */
   }
   ```

## 📝 技术细节

### Grid vs Flexbox

**为什么用 Grid 而不是 Flexbox？**

| 特性 | Grid | Flexbox |
|------|------|---------|
| 二维布局 | ✅ | ❌ |
| 自动换行 | ✅ | ⚠️ |
| 对齐控制 | ✅ | ⚠️ |
| 响应式 | ✅ | ⚠️ |

**Grid 优势**:
- 同时控制行和列
- 自动填充和适配
- 更好的对齐选项
- 原生响应式支持

### auto-fill vs auto-fit

```css
/* 使用 auto-fill */
grid-template-columns: repeat(auto-fill, minmax(600px, 1fr));

/* 如果使用 auto-fit */
grid-template-columns: repeat(auto-fit, minmax(600px, 1fr));
```

**区别**:
- `auto-fill`: 保持所有列，即使为空
- `auto-fit`: 折叠空列

我们选择 `auto-fill` 因为：
- ✅ 保持布局稳定
- ✅ 滚动时不会跳动
- ✅ 更好的视觉一致性
