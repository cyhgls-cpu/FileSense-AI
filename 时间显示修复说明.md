# 📅 文件时间显示修复说明

## 🐛 问题描述

**用户反馈**: "修改时间、创建时间都是未知，不能正确显示"

### 问题现象
```
📅 修改时间：未知
📏 创建时间：未知
```

---

## 🔍 根本原因

经过排查，发现问题在于：

### 1. **后端数据完整** ✅
```javascript
// src/main.js:113-114
files.push({
  path: fullPath,
  name: entry.name,
  size: stats.size,
  extension: ext,
  modifiedTime: stats.mtime.toISOString(),    // ✅ 正确保存
  createdTime: stats.birthtime.toISOString()  // ✅ 正确保存
});
```

### 2. **前端格式化正确** ✅
```javascript
// index.html:2063-2080
function formatDate(dateString) {
  if (!dateString) return '未知';
  
  try {
    const date = new Date(dateString);
    if (isNaN(date.getTime())) return '无效日期';
    
    return `${year}-${month}-${day} ${hours}:${minutes}:${seconds}`;
  } catch (err) {
    return '日期格式错误';
  }
}
```

### 3. **IPC 处理器缺失** ❌
```javascript
// 前端调用
duplicateFiles = await ipcRenderer.invoke('find-duplicates', scannedFiles);

// 后端只有
ipcMain.handle('find-duplicates-advanced', ...)
// ❌ 缺少 'find-duplicates' 处理器！
```

**结果**: 前端调用了不存在的 IPC 方法，导致返回 undefined 或错误

---

## ✅ 修复方案

### 添加缺失的 IPC 处理器

在 `src/main.js` 中添加：

```javascript
// 查找重复文件（基础版本，基于 MD5 哈希）
ipcMain.handle('find-duplicates', async (event, files) => {
  try {
    const duplicates = await findDuplicates(files);
    console.log(`[IPC] 找到 ${duplicates.length} 组重复文件`);
    
    // 验证时间信息是否存在
    if (duplicates.length > 0) {
      const firstGroup = duplicates[0];
      console.log('[IPC] 示例数据:', {
        originalPath: firstGroup.original.path,
        originalModified: firstGroup.original.modifiedTime,
        originalCreated: firstGroup.original.createdTime,
        duplicatePath: firstGroup.duplicate.path,
        duplicateModified: firstGroup.duplicate.modifiedTime,
        duplicateCreated: firstGroup.duplicate.createdTime
      });
    }
    
    return duplicates;
  } catch (err) {
    console.error('[IPC] 查找重复文件失败:', err);
    throw err;
  }
});
```

## 📝 可能的边界情况

### 情况 1: 文件系统不支持创建时间
**现象**: `birthtime` 返回 Unix 纪元时间 (1970-01-01)

**处理**: 
```javascript
if (stats.birthtime.getTime() === 0) {
  createdTime = stats.mtime.toISOString(); // 使用修改时间
}
```

### 情况 2: 网络驱动器
**现象**: 某些网络文件系统可能不提供完整的时间信息

**处理**: formatDate 函数会返回"无效日期"而不是崩溃

### 情况 3: 损坏的文件
**现象**: fs.statSync 可能失败

**处理**: 扫描时已经用 try-catch 包裹，跳过损坏的文件

## 💡 技术细节

### 为什么使用 toISOString()？

#### 优点：
1. **标准化**: ISO 8601 格式是国际标准
2. **无歧义**: 不受时区影响
3. **易解析**: JavaScript Date 可以直接解析
4. **可排序**: 字符串比较等于时间比较

#### 对比其他格式：
```javascript
// ❌ 不推荐：本地字符串
modifiedTime: stats.mtime.toString()
// "Sat Mar 14 2026 15:30:45 GMT+0800"
// 问题：格式复杂，依赖 locale

// ✅ 推荐：ISO 字符串
modifiedTime: stats.mtime.toISOString()
// "2026-03-14T15:30:45.123Z"
// 优点：标准、简洁、易解析
```

### 为什么前端要格式化？

ISO 字符串虽然标准，但对用户不友好：
```
❌ "2026-03-14T15:30:45.123Z"
✅ "2026-03-14 15:30:45"
```

formatDate 函数的作用就是转换为用户友好的格式。
