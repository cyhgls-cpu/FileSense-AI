## 🔧 技术架构

### 核心模块

```
src/
├── main.js              # Electron 主进程
├── scanner.js           # 文件扫描引擎 (启发式漏斗)
├── scanner-worker.js    # Worker 线程
├── index-db.js          # SQLite 索引数据库
├── ai-engine.js         # AI 引擎核心 ⭐新增
├── mft-scanner.js       # MFT 快速枚举 ⭐新增
├── mmap-reader.js       # 内存映射零拷贝 ⭐新增
├── bloom-filter.js      # 布隆过滤器 ⭐新增
└── batch-writer.js      # 批量事务写入 ⭐新增
```

### AI 模型栈

```
┌─────────────────────────────────────┐
│ 应用层：AI 功能调用                    │
├─────────────────────────────────────┤
│ 服务层：Embedding | CLIP | LLM       │
├─────────────────────────────────────┤
│ 推理层：ONNX Runtime | llama.cpp     │
├─────────────────────────────────────┤
│ 模型层：bge-micro | CLIP | Qwen2.5   │
├─────────────────────────────────────┤
│ 硬件层：CPU (AVX2) | GPU | NPU       │
└─────────────────────────────────────┘
```

## 🎯 使用场景演示

### 场景 1: 快速扫描 (极速模式)

```
用户操作:
  1. 打开应用 → 自动检测到旧电脑
  2. 推荐"极速模式" → 仅启用文档 AI
  3. 选择文件夹 → 开始扫描
  4. 3 秒后完成 10 万文件扫描
  
结果:
  ✅ 找到 45 组重复文件
  ✅ 发现 12 份语义相似的合同
  ✅ 内存占用仅 50MB
```

### 场景 2: 图片整理 (平衡模式)

```
用户输入: "找出所有发票和收据的照片"

AI 处理:
  1. CLIP 模型理解"发票"和"收据"的概念
  2. 搜索已预处理的图片库
  3. 返回 23 张匹配的图片
  
结果:
  ✅ 找到所有含发票特征的图片
  ✅ 即使拍摄角度不同也能识别
  ✅ 无需手动翻找
```

### 场景 3: 合同比对 (完全 AI 模式)

```
用户操作:
  1. 选择两个合同文件
  2. 点击"AI 智能分析差异"
  3. 等待 3 秒
  
LLM 输出:
  "v2 版本主要修改:
   1. 第三条第 2 款：金额从 10 万改为 12 万
   2. 新增违约责任条款 (第 8 条)
   3. 日期更新为 2024 年
   
   建议：保留 v2 版本，内容更完善"
```

## 📦 安装与依赖

### 基础安装 (必选)

```bash
cd "k:\AI soft\rerere"
npm install
npm start
```

### AI 功能 (可选)

```bash
# ONNX Runtime (文档 + 图片 AI)
npm install onnxruntime-node

# llama.cpp (LLM 智能分析)
npm install node-llama-cpp

# 图像处理
npm install sharp

# 向量搜索
npm install sqlite-vss
```

首次使用 AI 功能时自动下载：
