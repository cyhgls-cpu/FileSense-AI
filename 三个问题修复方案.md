### 问题 2: JPEG 预览错误

#### 根本原因
- sharp 库使用 libvips 处理 JPEG
- 某些 JPEG 文件使用了非标准的渐进式编码
- libvips 无法解析这些特殊格式的 SOS 参数

#### 修复策略：多层降级

```javascript
// src/main.js - get-image-thumbnail
async function getThumbnail(imagePath) {
  // 方案 1: 尝试使用 sharp（带 mozjpeg 提高兼容性）
  try {
    const sharp = require('sharp');
    const data = await sharp(imagePath)
      .resize(300, 300)
      .jpeg({ quality: 80, mozjpeg: true }) // mozjpeg 兼容性更好
      .toBuffer();
    return `data:image/jpeg;base64,${data.toString('base64')}`;
  } catch (sharpErr) {
    // sharp 失败，静默降级
    console.warn(`[Sharp] ${imagePath}: ${sharpErr.message}`);
  }
  
  // 方案 2: 直接读取原图 Base64（让浏览器处理解码）
  try {
    const fileData = fs.readFileSync(imagePath);
    const base64 = fileData.toString('base64');
    const mimeType = getMimeType(ext);
    return `data:${mimeType};base64,${base64}`;
  } catch (readErr) {
    console.warn(`[读取] ${imagePath}: ${readErr.message}`);
  }
  
  // 方案 3: 返回 null，前端隐藏 img
  return null;
}
```

#### 为什么不报错？

1. **VipsJpeg 错误是正常的**
   - 这是 libvips 的限制，不是 bug
   - 某些相机/软件生成的 JPEG 确实不标准
   
2. **我们优雅降级**
   - sharp 失败 → 直接读原图
   - 原图失败 → 隐藏预览图
   - 不影响文件整理功能

3. **减少日志噪音**
   ```javascript
   if (!err.message.includes('VipsJpeg') && 
       !err.message.includes('Invalid SOS')) {
     console.warn(`[Sharp] ${imagePath}: ${err.message}`);
   }
   // 已知问题只记录警告，不报错
   ```

#### 支持的 JPEG 类型

| 类型 | sharp | 原图直读 | 最终方案 |
|------|-------|---------|---------|
| 标准 JPEG | ✅ | ✅ | sharp |
| 渐进式 JPEG | ⚠️ 可能失败 | ✅ | 原图直读 |
| 损坏 JPEG | ❌ | ⚠️ 可能 | 隐藏 |
| HEIC/HEIF | ❌ | ❌ | 隐藏 |

**步骤 4: 检查格式化函数** ✅
```javascript
// index.html:2063-2080
function formatDate(dateString) {
  if (!dateString) return '未知';
  
  try {
    const date = new Date(dateString);
    if (isNaN(date.getTime())) return '无效日期';
    
    return `${year}-${month}-${day} ${hours}:${minutes}:${seconds}`;
  } catch (err) {
    return '日期格式错误';
  }
}
```

**如果时间仍显示"未知"，检查**:

2. 控制台是否有 `[前端] 第一组数据` 日志？
   - 没有 → 前端代码未执行，检查 index.html 是否正确
   - 有但 hasModified: false → 后端返回的数据缺少该字段

3. formatDate 函数是否被调用？
   ```javascript
   // 在 formatDate 中添加日志

     // ...
   }
   ```

#### 常见陷阱

**陷阱 1: 代码未重新加载**
```bash
# 修改后必须重启应用
npm start
# 或双击 一键启动.cmd
```

**陷阱 2: 缓存问题**
```bash
# 清除缓存
Ctrl+Shift+R 强制刷新
# 或关闭应用重新打开
```

**陷阱 3: 文件系统不支持**
- 某些网络驱动器不提供 birthtime
- FAT32/exFAT 文件系统可能只有 modifiedTime

解决方案：
```javascript
// 如果 createdTime 无效，使用 modifiedTime
if (!stats.birthtime || stats.birthtime.getTime() === 0) {
  createdTime = stats.mtime.toISOString();
} else {
  createdTime = stats.birthtime.toISOString();
}
```
